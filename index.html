Basket

API that allows to retrieve and modify contents of basket for user.

Each user has only one basket, that can be used either to place orders, or create new subscriptions - both actions will clear basket afterwards.

It is important to note that while basket quantities are limited by warehouse, they are not guaranteed, as adding product to basket does not make product reserved for specific user - quantities can be force-changed during modification calls when warehouse quantity shrinks due to other users placing orders, and final quantities are evaluated while processing order placement.

ğŸ” Get basket contents

Returns basket summary, as well as product listing with selected quantities.

Detail level of basket can be tweaked using flags in query params.

Allows forcing different month prices using X-* headers.

Product list will contain special product with productId == 0, which will be delivery.

Subscriptions

API that allows creating, retrieving, modifying and deleting subscriptions for user.

ğŸ” Create subscription

Creates subscription based on current basket contents.

Created subscription is not filled with order details and is inactive.

Unlike basket, it does not coerce quantities into available stock and will error if requested quantities are not in stock, however unlike order, it does not immediately take product from warehouse - subscription will be translated to order when its frequency ticks (subscription must be active), and only then quantities will be evaluated.

ğŸ” Get subscription data

Returns full data of subscription.

Prices of products will only corelate to current prices, not the ones that will be used in final order.

Product list will contain special product with productId == 0, which will be delivery. Product quantities only reflect quantities specified when subscription was created and are not updated to show current availability.

ğŸ” Update subscription

Allows to update order details for subscription.

Does not allow alterations to the product list.

It is important to note, that subscription being active implies that order details (address) is filled. Attempting to activate subscription either without it having address set prior, or without sending address with activation signal (simultaneous updates might not be present on frontend), or event trying to remove address without deactivating subscription, will error.

Authorizations:
(jwtBearerCookiejwtRefreshCookie)
Request Body schema: application/json
required

Request body for this endpoint must contain at least one property.

status
string
Enum: "Active" "Paused"

Requests new status of subscription.

addressName:
string

Name of address to be set for order details. If this parameter has value of empty string, address will be cleared instead.
